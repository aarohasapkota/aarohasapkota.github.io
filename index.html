üìå Project Overview

This is a full chess game built in Java. It supports local multiplayer, online games with friends, and an AI opponent using the Stockfish engine. The app follows MVC, has a JavaFX GUI, and uses Firebase for user accounts, profiles, and a friends system with real-time updates.

Purpose

The goal was to build a full object-oriented chess application from scratch, practice design patterns, and connect a real chess engine and online features. It's meant to feel like a complete chess app, not just a small school demo.

Technology Stack

  • Java (game logic + backend)
  • JavaFX (GUI with board and screens)
  • Firebase (auth + real-time data for users and friends)
  • Stockfish Chess Engine (AI opponent)
  • Maven project structure
  • JUnit + JavaDoc
  • MVC architecture

Design & Architecture

The project uses a clear model layer for pieces, board, and game state. FEN notation is used for board serialization. The code follows MVC and uses several design patterns:

  • Decorator ‚Äì chain move validation checks
  • Observer ‚Äì update UI and other parts when the game state changes
  • Template Method ‚Äì support different game modes (local, online, vs AI)
  • Adapter ‚Äì connect to the Stockfish engine

Testing & Documentation

I wrote JUnit tests for core game logic like piece movement, coordinates, and move validation. I also generated JavaDoc (100+ HTML pages) for all major classes and methods.

üìú Move History & Stats

üìò Use Cases

Play Local Game Use Case

  1. User logs in or continues as a local player.
  2. User chooses "Local Game" from the main menu.
  3. User picks colors and time settings (if needed).
  4. The system creates a new game and shows the chess board.
  5. Both players take turns moving pieces on the same machine.
  6. At the end, the result is stored and stats are updated.

Play Online Game Use Case

  1. User logs in with their Firebase account.
  2. User opens the Friends / Online Game section.
  3. User invites a friend or accepts an invite.
  4. The system sets up a shared game state using Firebase.
  5. Both players see moves update in real time.
  6. Game result is saved and shown in history and stats.

Play vs AI Use Case

  1. User chooses "Play vs AI" mode.
  2. User picks color and time settings.
  3. The system starts a game and connects to Stockfish.
  4. User makes a move; the Adapter sends the position to Stockfish.
  5. Stockfish returns a move, and the game state updates.
  6. Result is saved and can be viewed later.
